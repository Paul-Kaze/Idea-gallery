# FRD: ImageDetailModal 自适应与固定尺寸优化

## 1. 概述 (Overview)
本次优化的目标是提升 `ImageDetailModal`（图片/视频详情弹窗）组件的布局稳定性和浏览体验。通过固定左侧媒体（图片和视频）展示区域的大小，并使用自适应缩放机制，确保各种比例的媒体资源在展示时不会导致弹窗尺寸突变。同时，右侧的信息展示面板将支持独立滑动，确保长文本内容的展示不会撑大整个弹窗组件，从而保证一致且优雅的用户界面。

## 2. 功能需求明细 (Functional Requirements)

| 一级功能 | 二级功能 | 三级功能 | 需求具体描述 | 优先级 |
|---------|---------|---------|------------|-------|
| 详情弹窗 | 媒体展示区 | 尺寸固定 | 媒体展示区域需设定固定的宽高比例或最大尺寸，不随内部媒体资源的实际分辨率发生变化。 | P0 |
| 详情弹窗 | 媒体展示区 | 自适应按比例缩放 | 在固定尺寸的展示区内，图片和视频需按完全展示自身原本比例进行等比缩放（如 CSS 的 `object-fit: contain`），避免画面拉伸变形，同时防止资源过大撑出展示区边界。 | P0 |
| 详情弹窗 | 信息展示区 | 尺寸限制 | 右侧的信息展示面板需设定高度限制（例如与左侧媒体区域等高），不得超过整个弹窗组件所划定的边界。 | P0 |
| 详情弹窗 | 信息展示区 | 内部独立滚动 | 当信息面板的内容（如标题、描述、各项参数信息等）超出面板自身高度时，面板内部应支持垂直滚动/滑动查看。 | P0 |
| 详情弹窗 | 整体布局 | 尺寸稳定性 | 弹窗（`ImageDetailModal`）的整体外部尺寸大小在加载不同宽高比媒体资源、或展示不同长度文本信息时应保持固定不变，避免影响背景遮罩及页面整体重排。 | P0 |

## 3. 错误处理与用户提示 (Error Handling & User Feedback)
- **ERR-001:** 当媒体资源（图片/视频）加载失败或损坏时 -> 在固定的媒体展示区域内居中显示错误提示（例如错误图标或“媒体加载失败”占位图），整个区域保持原本规划好的固定尺寸，不允许塌陷。
- **ERR-002:** 当右侧信息由于网络延迟等原因未获取到时 -> 右侧面板结构保持固定占比宽度和高度，内部可使用骨架屏或“暂无信息”提示，不能因为无内容而导致左右布局比例失衡。

## 4. 数据与交互细节 (Data & Interaction Details)
- **布局交互逻辑：** 当用户在右侧信息面板内进行上下滚动交互时，左侧的媒体展示区应保持静止锁定，两侧区域互不干扰。
- **响应式适配：** 在移动端屏幕中（若设计稿变为上下布局结构），原右侧的信息面板应变为下方独立滑动的面板，同样不能因此撑高弹窗超过屏幕总高度，左/上的媒体区需按设备宽度维持同样的固定比例计算方式。
